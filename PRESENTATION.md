## 1. 개발 목적

**해결하려는 문제**
- 구독 중인 유튜브 채널의 신규 영상이 매일 쌓이지만 다 볼 시간은 없다.
- 어떤 영상에 시간을 투자할지 빠르게 판단하기 어렵다.
- 유튜브 기본 자막은 길고 비구조화 텍스트라 훑어 읽기에 비효율적이다.

**서비스 목표**
- 구독 채널의 신규 영상을 **자동 감지** → **AI가 구조화된 요약**으로 변환 → **대시보드에 아카이빙**.
- 단순 한두 줄 캡션이 아니라 **헤드라인·본문·출연자·언급 기업·주제·타임스탬프**까지 분리된 데이터로 저장.
- 영상 한 편을 1분 안에 파악, 필요한 부분만 골라서 시청.

## 2. 주요 기능

| 영역 | 기능 |
|---|---|
| **자동 수집** | 채널 ID로 YouTube RSS 피드를 주기 폴링, 신규 영상 자동 감지 (중복 방지) |
| **자막 추출** | `youtube-transcript`로 한국어/영어 자막 자동 추출. 실패 시 영상 설명문(메타데이터)로 폴백해 파이프라인이 멈추지 않음 |
| **AI 요약** | OpenAI `gpt-5-mini`로 JSON 스키마에 맞춰 구조화 — headline / brief(2,000자 산문) / topics / people / companies / 타임스탬프 |
| **출연자 ↔ 언급 인물 분리** | LLM 이 "직접 출연" vs "대화 중 거론" 을 별도 필드로 분리. 역사적 인물·인용된 학자·사례 속 인물은 `mentioned_people` 로 정렬 (예: 곽재식 회차에서 곽재식 = people, 이호왕·김수암 = mentioned_people) |
| **출연자 정보** | 영상 설명문에서 학력·저서 등 약력까지 자동 추출 (음성 인식 오류도 설명문 우선으로 정정) |
| **기업 추출** | 본문에 등장한 기업·연구기관을 별도 필드로. 광고·시청 안내·지수명 등 6가지 노이즈 패턴 자동 차단 |
| **코너 단위 구독 (제목 + 설명 키워드)** | 한 채널에 여러 코너가 있을 때 키워드 AND 필터로 원하는 코너만 매치. 매처가 **제목 + RSS 설명문**을 함께 검색해서 호스트가 제목에 안 적힌 채널 (예: 이강민의 잡지사 → `#곽재식` 해시태그가 설명문에만 있음) 도 잡힘 |
| **채널 관리 UI** | `/channels` 페이지에서 폼으로 채널/구독 등록·삭제. 서버 액션 사용해 비밀 키 노출 없이 호출 |
| **대시보드** | 채널 칩 row (신규 영상 NEW 뱃지) + 영상 카드 그리드 + 상세 페이지 (타임스탬프 → 유튜브 점프 링크, 통합 키워드 섹션) |
| **자동화** | Vercel Cron 일 1회 — `refresh` → `process` → `summarize` 시퀀스 자동 실행. `channelId` 옵션으로 특정 채널 우선 처리도 가능 |

## 3. 기술 스택

### 프레임워크 / 언어
- **Next.js 16** (App Router) — 풀스택 React 프레임워크. API Routes로 백엔드까지 통합
- **TypeScript** — 타입 안전성
- **Tailwind CSS + shadcn/ui** — 디자인 시스템

### 데이터 / 모델
- **Supabase (PostgreSQL)** — 채널·구독·영상·자막·요약 저장. Service Role Key로 서버 사이드 접근
- **OpenAI gpt-5-mini** (현재) — 구조화된 JSON 출력 (response_format: json_schema), 한국어 품질·비용 균형
- **OpenAI gpt-4o-mini** (이전 단계) — 초기 MVP에서 사용. 영상 1편당 비용은 약 1/5로 저렴하지만 brief 분량·문단·타임스탬프 규칙을 잘 안 지키고 누락이 잦아 5-mini로 업그레이드
- (비교 실험) **Claude Sonnet 4.6** — 결과는 거의 동등했으나 비용/안정성에서 gpt-5-mini 채택

### 외부 데이터 소스
- **YouTube RSS** (`feeds/videos.xml?channel_id=…`) — 채널당 최근 15개 영상. API 키 불필요
- **YouTube 영상 페이지 스크래핑** — 제목·설명문·channel_id 추출
- **youtube-transcript** 라이브러리 — 자동 생성 자막 (한/영 fallback)

### 배포 / 운영
- **Vercel** (Hobby 플랜) — Next.js native, Serverless Functions
- **Vercel Cron** — 일 1회 통합 작업 트리거
- **GitHub** — 소스 관리 + Vercel 자동 빌드 연동

**Vercel Hobby 플랜 용량 제한 (감안한 설계 제약)**
- Cron Job: 프로젝트당 **최대 2개**, 각 cron 호출 빈도 **일 1회**
- Serverless Function 실행 시간: **최대 60초**
- 월 함수 호출: 100K 회, 대역폭 100GB (현 규모에서는 여유)

→ 위 제약을 반영해 `/api/cron` 통합 엔드포인트 1개로 묶고, 한 번 호출에 영상 1~2개만 처리 (잔여는 다음 cron 또는 수동 호출). 운영 확장 시 Pro 플랜 업그레이드로 매시간 cron + 300초 timeout 활용 가능.

### 보안
- API 키는 `.env.local` 서버 전용. 클라이언트 노출 금지
- 모든 API Route에 `CRON_SECRET` Bearer 인증 (외부 무단 호출 차단)

## 4. 진행 상황

### 완료된 항목

**파이프라인**
- ✅ RSS 신규 감지 → 자막 추출 → 메타데이터 폴백 → LLM 구조화 요약 → DB 저장 전 과정 작동
- ✅ 검증·재시도 루프: brief 분량/단락/타임스탬프 미달 시 자동 재호출
- ✅ 영상 9건 실제 요약 검증 — 김작가 TV / 신사임당 / EBS / PLUS TV / KBS 생로병사 / 한경 글로벌마켓 / MBC 라디오 시사 등

**요약 품질 반복 개선**
- 초기 결과: brief 396자, 1문단, 출연자 자리에 기업명 9개 환각, 위반 3건
- 최종 결과: brief 2,000자대, 9문단, 타임스탬프 10~46개, 위반 0건, 출연자·기업 정확
- 음성 인식 오류 자동 정정 사례 — 자막 "한재원" / "젠슨왕" → 모델이 설명문 근거로 "한재권" / "젠슨 황"으로 정정

### 🎯 옛 규칙 vs 새 규칙 — 라이브 비교 (발표 데모용)

**같은 채널(Normaltic Place) 안에 두 영상을 의도적으로 보존해 비교 가능하도록 둠.**

| 영상 | URL | 적용 규칙 |
|---|---|---|
| `8Khu3IoZric` (다크웹·토르) | https://tubebrief-mvp.vercel.app/videos/8Khu3IoZric | **옛 규칙** (초기 5단계 강화 프롬프트) |
| `muA-MB0k7SE` (인터넷 검열의 역사) | https://tubebrief-mvp.vercel.app/videos/muA-MB0k7SE | **새 규칙** (mentioned_people·companies·키워드 통합·시각 마커 검증) |

**한눈에 보이는 차이**

| 항목 | 옛 규칙 8Khu3IoZric | 새 규칙 muA-MB0k7SE |
|---|---|---|
| 자막 자료 길이 | 463자 (1분 영상) | 13,641자 (30분 영상) |
| headline | 25자 | 85자 |
| **brief 분량** | **366자 1문단** | **2,122자 10문단** |
| `(mm:ss)` 시각 마커 | **0개** | **5개** (00:00·02:09·05:39·19:09·25:46) |
| companies (기업 추출) | 0개 | 7개 (구글·클라우드플레어·노드VPN·텔레그램·유튜브·카카오톡·네이버) |
| topics (주제) | 5개 | 6개 |
| 출연자/거론 인물 분리 | 한 필드 혼재 (옛 schema) | `people` / `mentioned_people` 분리 |
| UI 키워드 표시 | "주제" / "기업" 별도 섹션 (옛) | **통합 "키워드" 섹션** (새) |

**무엇이 개선됐나 — 5가지 규칙 강화**

| 어려움 | 첫 시도 결과 | 강화된 규칙 | 결과 변화 |
|---|---|---|---|
| LLM이 한국어 요약 분량 부족 | brief 290자 1문단 평균 | 자료 크기별 하한선 + 검증/재시도 루프 + 모델 교체 (gpt-4o-mini → gpt-5-mini) | brief 1,800~2,200자 8~10문단 |
| 출연자와 거론된 인물 혼재 | 한 필드에 모두 (예: 곽재식+이호왕+김수암) | `people` / `mentioned_people` 분리 + 판별 기준 ("직접 말하나? vs 이름만 거론?") | 영상에 직접 출연한 사람만 출연자, 나머지는 거론 인물 |
| 시각 마커 무시 | 0개 / 영상 | "최소 2~5곳 적극 삽입" 명령조 + 사후 검증으로 자동 재시도 | 영상당 평균 (mm:ss) 5개 |
| 기업·주제 한 덩어리 | topics 만 있음 | `companies` 별도 필드 + 6규칙 노이즈 차단 (광고·OTT·지수 등 제외) | 기업 정확 추출 + 환각 방지 |
| UI 시각 분산 | "주제" + "기업" 별도 섹션 | 사용자 인지 부담 ↓ 위해 통합 "키워드" 섹션으로 합침 | 시각적 일관성 ↑ |

**노이즈 차단 6규칙 (companies 필드)**
1. 사람 외형 묘사 ("웨이브 머리"의 '웨이브')
2. OTT/시청 안내 (티빙·왓챠 등)
3. 정부 부처·대회명 (DARPA 챌린지·CES 등)
4. 지수·종목군 (S&P 500·KOSPI·M7 등)
5. 영상 게시 채널 (KBS·EBS·MBC 자체)
6. 협찬·제작도움 (KB자산운용 등)

**구독 시스템**
- ✅ 채널 ↔ 영상 사이에 `subscriptions` 레이어 도입
- ✅ include/exclude 키워드 AND 필터로 한 채널 안의 특정 코너만 매치
- ✅ 실측 검증: MBC 채널 15개 영상 중 `[뉴스하이킥] + 곽상준` 조건으로 1개만 정확히 매치

**UI**
- ✅ 메인 페이지 채널 칩 (영상 수, NEW 뱃지, 클릭 시 채널 필터링)
- ✅ 영상 카드 그리드 + 상세 페이지
  - 상세: 헤드라인 / 통합 **키워드** (주제 + 기업) / 출연자 / **언급된 인물** / 본문 brief (타임스탬프 → 유튜브 점프 링크)
- ✅ **`/channels` 채널 관리 페이지** — 폼으로 채널/구독 등록 (URL + 라벨 + include/exclude 키워드), 2단계 컨펌 삭제
- ✅ 한국어 폰트(Noto Sans KR), 에러·로딩·404 페이지
- ⚠️ 향후 보완 후보: 검색·필터, 모바일 최적화, v0.dev 기반 디자인 재설계

**배포 준비**
- ✅ `vercel.json` (Hobby 플랜 제약 반영 — 일 1회 cron, 60초 timeout)
- ✅ `/api/cron` 통합 엔드포인트
- ✅ API 인증 (`CRON_SECRET` Bearer)
- ✅ 빌드 검증 통과

### 비용 / 성능 지표
- 영상 1편당 토큰: 평균 약 20,000 (입력 15k + 출력 5k)
- 영상 1편당 비용: 약 **$0.025** (gpt-5-mini 기준)
- 자막 23,000자대 긴 영상(45분)도 단일 호출로 처리 성공
- 자동화 후 월 운영 비용 예상: 영상 100편 처리 시 약 **$2.5/월**

### 남은 작업
- GitHub push → Vercel import → 환경변수 등록 → Deploy
- 첫 자동 cron 실제 동작 모니터링

### 한계와 향후 개선
- **RSS 의 "최근 15개" 제약** — 채널의 그 이전 영상은 RSS 에서 사라져 못 잡음. 한계 인정. 보완하려면 YouTube Data API 도입 필요 (쿼터 + 인증 부담).
- **자막 음성 인식 오류** — "곽재식" 이 "곽제식" 으로 음역됨. 현재는 설명문 (description) 의 호스트 이름 표기를 LLM 한테 정답 표기로 알려주는 방식을 검토 중. (선택 개선)
- **Vercel Hobby 의 60초 / 일 1회 cron 제약** — `/api/cron` 1회당 영상 1~2개씩만 처리. backlog 가 쌓이면 따라잡는 데 시간 걸림. 운영 확장 시 Pro 업그레이드 ($20/월) 또는 외부 워커로 분리 검토.
- **인증 부재** — 현재 배포 URL 은 공개. 채널 추가/삭제도 누구나 가능. Vercel Password Protection 또는 Supabase Auth 단계 필요 (현재는 MVP 단일 사용자).
- **검색·필터·텍스트 내보내기** — 영상이 더 쌓이면 필요. v2 개선 후보.
